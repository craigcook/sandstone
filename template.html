Mozilla Sandstone

A global style for Mozilla websites

What is Sandstone

Sandstone is our defined style and form for Web properties. Combining set visual styles, UX elements, and a responsive grid system, it creates a cohesive brand experience across all our websites. You can learn more about it in the Mozilla style guide.

Responsive grids

The default grid system for Sandstone is 12 columns at a width of 940px, where each column is 60px wide with 20px gutters. Those gutters are achieved with 10px margins on the left and right of the columns so elements can span columns and float next to each other in an endless variety of configurations.

The columns switch to 40px wide for smaller viewports, those between 760px and 480px wide. For especially small viewports – namely smartphones or other handheld mobile devices – the grid drops to 5 columns at 40px and most content is allowed to stack vertically rather than float side by side. Bear this in mind when you arrange your content so it can maintain a sensible flow in both configurations.

Headings

The secondary heading above is an h2 element, which may be used for any form of important page-level heading. Primary titles, such as the page title or the title of this article can be h1 elements. In HTML5, each new section element in a document establishes a new context with a new heading hierarchy, so it's perfectly acceptable to have more than one h1 on a single page, but there should generally be as few as possible.

Use headings in the correct order to indicate hierarchy and the relative importance of the content they introduce.

Sidebar

Some pages may have optional sidebars to contain supplemental content or navigation. Sidebars typically wrap under the main column in mobile viewports, and should always be considered secondary to the main intent of the page. If a sidebar contains vital and essential content, then perhaps that content shouldn't be in a sidebar.
